# k6 rendimiento

El proyecto incluye pruebas de rendimiento con k6 para evaluar el comportamiento del backend bajo distintos escenarios de carga.

## Ubicación

```text
k6/
├─ tests/
│  ├─ 01_smoke_test.js
│  ├─ 02_load_test.js
│  ├─ 03_stress_test.js
│  ├─ 04_spike_test.js
│  └─ 05_soak_test.js
├─ config/
├─ grafana/
├─ run.sh
├─ run.ps1
└─ docker-compose.yml
```

## Tipos de prueba

| Prueba | Objetivo |
|---|---|
| Smoke test | Verificar que el sistema responda con carga mínima. |
| Load test | Evaluar comportamiento bajo carga esperada. |
| Stress test | Evaluar límites bajo carga alta. |
| Spike test | Evaluar respuesta ante picos repentinos. |
| Soak test | Evaluar estabilidad durante un periodo prolongado. |

## Ejecución referencial

Según el entorno, se puede ejecutar mediante scripts o Docker Compose.

```bash
cd k6
./run.sh
```

En Windows PowerShell:

```bash
cd k6
.\run.ps1
```

## Evidencia esperada

- Tiempo promedio de respuesta.
- Percentiles de respuesta.
- Tasa de errores.
- Solicitudes por segundo.
- Reportes o dashboards de ejecución.

!!! note "Contexto académico"
    Las pruebas de rendimiento son útiles para validar técnicamente el backend, pero no reemplazan una prueba de capacidad formal en producción.
