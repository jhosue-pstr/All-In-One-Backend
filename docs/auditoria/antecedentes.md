# Antecedentes

Los antecedentes explican qué es All-InOne y por qué tiene sentido auditarlo.

## Descripción del sistema

All-InOne es una plataforma SaaS multitenant orientada a pequeñas y medianas empresas. Permite crear sitios web, administrar contenido y activar módulos de negocio por sitio.

## Arquitectura y tecnología

El sistema combina:

- Frontend con React, Vite y TypeScript.
- Backend con FastAPI y Python.
- Persistencia mediante SQLAlchemy y base de datos relacional.
- Constructor visual integrado con GrapesJS.
- Autenticación JWT.
- Roles y permisos.
- Auditoría y soft delete.
- Herramientas DevSecOps como Jira, Jenkins, Sonar, Snyk, Playwright y k6.

## Contexto de auditoría

Debido a la complejidad del sistema, la auditoría busca verificar la coherencia entre documentación, arquitectura, implementación, pruebas y criterios de calidad.
