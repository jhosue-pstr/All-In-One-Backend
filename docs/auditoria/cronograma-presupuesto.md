# Cronograma y Presupuesto

## Cronograma

El cronograma diferencia dos periodos importantes:

| Periodo | Descripción |
|---|---|
| Desarrollo auditado | Periodo en el que se desarrolló All-InOne mediante sprints |
| Auditoría | Periodo estimado para revisar evidencias, evaluar y emitir resultados |

## Línea base del desarrollo

El desarrollo se organizó en sprints:

| Sprint | Enfoque |
|---|---|
| Sprint 0 | Planificación y arquitectura |
| Sprint 1 | Core Platform y gestión de sitios |
| Sprint 2 | Constructor visual |
| Sprint 3 | Módulo Blog |
| Sprint 4 | Módulo Tienda |
| Sprint 5 | Auth Público |
| Sprint 6 | Plantillas reales |
| Sprint 7 | Inventario, Analítica y cierre |

## Fases de auditoría

| Fase | Actividad principal |
|---|---|
| Fase 1 | Preparación y planificación |
| Fase 2 | Revisión documental y trabajo de campo |
| Fase 3 | Evaluación y reporte |
| Fase 4 | Seguimiento y cierre |

## Presupuesto estimado

El presupuesto es referencial y académico. No representa un pago real, sino una valorización del esfuerzo auditor.

| Recurso | Costo estimado |
|---|---:|
| Auditor líder | S/ 800.00 |
| Auditor de software / arquitectura | S/ 720.00 |
| Especialista QA y pruebas | S/ 504.00 |
| Analista de procesos y documentación | S/ 480.00 |
| Herramientas y software de apoyo | S/ 100.00 |
| Materiales y documentación | S/ 70.00 |
| Imprevistos | S/ 214.00 |
| **Total estimado** | **S/ 2,888.00** |

!!! warning "Importante"
    Este presupuesto no representa una obligación de pago. Es una simulación razonable del costo equivalente del trabajo auditor.
