# Criterios y Metodología de Auditoría

## Criterios de auditoría

La auditoría no se basa solo en opiniones. Se apoya en criterios técnicos y buenas prácticas.

| Criterio | Uso en la auditoría |
|---|---|
| ISO/IEC 12207 | Evaluar procesos del ciclo de vida del software |
| ISO/IEC 25010 | Evaluar calidad del producto software |
| CMMI-DEV | Revisar madurez de procesos de desarrollo |
| Checklist SDLC | Aplicar preguntas concretas de verificación |
| Desarrollo seguro | Revisar autenticación, permisos, validación y multitenancy |
| DevSecOps | Revisar GitHub, Jenkins, Sonar, Snyk, Playwright, k6 y Swagger |

## Metodología

La auditoría se ejecuta en cuatro fases.

### Fase 1: Preparar y planificar

Se definen objetivos, alcance, criterios, evidencias requeridas, plan de auditoría y checklist SDLC.

### Fase 2: Revisar y describir el proceso

Se revisa la documentación, Jira, backend, frontend, pruebas, seguridad, calidad, módulos y evidencias técnicas.

### Fase 3: Evaluar y reportar

Se comparan las evidencias contra los criterios definidos. Se identifican conformidades, no conformidades, brechas, riesgos y recomendaciones.

### Fase 4: Seguimiento

Se revisan acciones correctivas, responsables, evidencias de atención y cierre formal de la auditoría.

!!! info "Enfoque"
    La auditoría es evaluativa, no correctiva. Identifica hallazgos y recomendaciones, pero no modifica directamente el código fuente.
