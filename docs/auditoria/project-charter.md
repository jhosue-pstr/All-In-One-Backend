# Project Charter de Auditoría SDLC

El **Project Charter** es el documento de inicio formal de la auditoría SDLC del proyecto **All-InOne**.

Su función es dejar claro qué se va a auditar, por qué se audita, con qué criterios, bajo qué alcance, con qué metodología y qué entregables se esperan.

## Información general

| Campo | Detalle |
|---|---|
| Proyecto auditado | All-InOne |
| Organización auditada | ModularSoft - Grupo 4 |
| Tipo de auditoría | Auditoría del Ciclo de Vida del Desarrollo de Software |
| Código | AUD-SDLC-ALL-INONE-2026-001 |
| Contexto | Académico / Ingeniería de Software II |
| Clasificación | Confidencial - uso interno académico |

## Propósito del documento

El Project Charter permite formalizar la auditoría y establecer una base común para el equipo auditor, el profesor responsable, el patrocinador y el equipo auditado.

En términos simples, responde a estas preguntas:

- ¿Qué sistema se audita?
- ¿Por qué se audita?
- ¿Qué partes serán revisadas?
- ¿Qué queda fuera del alcance?
- ¿Qué criterios se usarán?
- ¿Cómo se ejecutará la auditoría?
- ¿Qué documentos se entregarán?
- ¿Qué riesgos deben controlarse?

## Estructura del Project Charter

| Sección | Contenido |
|---|---|
| Información general | Datos principales de la auditoría |
| Antecedentes | Descripción del sistema y contexto |
| Justificación | Razones por las que se realiza la auditoría |
| Objetivos | Objetivo general y objetivos específicos |
| Alcance y exclusiones | Qué se revisa y qué no se revisa |
| Criterios | ISO/IEC 12207, ISO/IEC 25010, CMMI-DEV, checklist SDLC y DevSecOps |
| Metodología | Fases de preparación, revisión, evaluación y seguimiento |
| Entregables | Documentos que generará la auditoría |
| Riesgos | Posibles problemas que pueden afectar la auditoría |
| Cierre | Supuestos, restricciones, aceptación y aprobaciones |

!!! tip "Idea clave para exposición"
    El Project Charter no es el informe final de auditoría; es el documento que autoriza y organiza la auditoría antes de ejecutarla.
