# Defensa en exposición

Esta página resume respuestas útiles para sustentar la auditoría.

## Preguntas frecuentes

### ¿Por qué hicieron una auditoría si el proyecto es académico?

Porque el proyecto integra varias funcionalidades técnicas y documentales. La auditoría permite validar si el desarrollo fue ordenado, trazable y coherente.

### ¿Por qué algunas partes del checklist están en No Cumple?

Porque no se debe forzar cumplimiento sin evidencia. Algunos ítems exigen formalidad empresarial, producción real, autorizaciones o mantenimiento formal, y el proyecto es académico/de prueba.

### ¿MkDocs cuenta como evidencia?

Sí, cuenta como evidencia documental técnica. Ayuda a organizar arquitectura, instalación, API, pruebas, seguridad y auditoría SDLC.

### ¿MkDocs reemplaza el Project Charter?

No. MkDocs publica y organiza la información; el Project Charter sigue siendo el documento formal de inicio de auditoría.

### ¿La auditoría corrige el sistema?

No. La auditoría identifica hallazgos, riesgos y recomendaciones. Las correcciones corresponden al equipo ModularSoft.

### ¿El despliegue de prueba equivale a producción?

No. Sirve para validación técnica, pero no representa implementación oficial en una empresa real.
