# Criterios de auditoría

Los criterios de auditoría son las referencias contra las que se evaluará el proyecto.

## Criterios utilizados

| Criterio | Uso en la auditoría |
|---|---|
| ISO/IEC 12207 | Evaluar procesos del ciclo de vida del software. |
| ISO/IEC 25010 | Evaluar calidad del producto software. |
| CMMI-DEV | Revisar madurez del proceso de desarrollo. |
| Checklist SDLC | Aplicar preguntas de verificación. |
| Buenas prácticas de desarrollo seguro | Revisar autenticación, autorización y protección de datos. |
| DevSecOps | Revisar GitHub, Jenkins, Sonar, Snyk, Playwright, k6 y Swagger. |

## Por qué no basta un solo criterio

Cada criterio evalúa una dimensión distinta. Por eso se combinan estándares, checklist y buenas prácticas.

!!! tip "Frase de defensa"
    Los criterios evitan que la auditoría se base en opiniones; permiten una evaluación objetiva y sustentada.
