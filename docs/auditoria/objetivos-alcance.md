# Objetivos y alcance

## Objetivo general

Evaluar formalmente el ciclo de vida del desarrollo de software de All-InOne para determinar su cumplimiento frente a estándares y buenas prácticas SDLC.

## Objetivos específicos

Los objetivos específicos organizan la auditoría por áreas:

| Objetivo | Área evaluada |
|---|---|
| OE1 | Gestión del proyecto. |
| OE2 | Requisitos. |
| OE3 | Diseño y arquitectura. |
| OE4 | Construcción y codificación. |
| OE5 | Pruebas. |
| OE6 | Implementación y despliegue. |
| OE7 | Mantenimiento y soporte. |
| OE8 | Documentación. |
| OE9 | Seguridad. |
| OE10 | Calidad del producto. |

## Alcance

La auditoría revisa gestión, requisitos, diseño, desarrollo, pruebas, implementación, mantenimiento, documentación, seguridad y calidad.

## Exclusiones importantes

No incluye certificación oficial, pentesting profundo, revisión línea por línea, corrección de código ni validación legal/financiera.
