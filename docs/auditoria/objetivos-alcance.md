# Objetivos, Alcance y Exclusiones

## Objetivo general

Realizar una auditoría formal del Ciclo de Vida del Desarrollo de Software de All-InOne, evaluando gestión, requisitos, arquitectura, implementación, pruebas, seguridad, documentación y calidad.

El propósito es identificar cumplimiento, brechas, riesgos y recomendaciones de mejora.

## Objetivos específicos

La auditoría se organiza en diez objetivos específicos:

| Objetivo | Área evaluada |
|---|---|
| OE1 | Metodología y gestión del proyecto |
| OE2 | Análisis de requisitos |
| OE3 | Diseño del sistema |
| OE4 | Construcción y codificación |
| OE5 | Pruebas |
| OE6 | Implementación y despliegue |
| OE7 | Mantenimiento y soporte |
| OE8 | Documentación |
| OE9 | Seguridad |
| OE10 | Calidad del producto |

## Alcance

La auditoría revisa los principales aspectos del SDLC:

- gestión del proyecto en Jira;
- requisitos e historias de usuario;
- arquitectura de monolito modular;
- backend y frontend;
- pruebas funcionales, E2E, rendimiento y seguridad;
- documentación técnica;
- controles de seguridad;
- calidad del producto;
- evidencias de DevSecOps.

## Exclusiones

La auditoría no incluye:

- auditoría financiera;
- evaluación comercial;
- validación legal;
- certificación oficial ISO;
- pentesting avanzado;
- modificación directa del código;
- revisión exhaustiva línea por línea;
- garantía de operación en producción;
- evaluación de clientes reales o datos productivos.

!!! summary "Idea clave"
    El alcance indica lo que sí se revisa; las exclusiones indican lo que no se promete revisar.
