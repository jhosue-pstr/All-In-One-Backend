# Auditoría SDLC

Esta sección reúne la documentación de auditoría del Ciclo de Vida del Desarrollo de Software del proyecto **All-InOne**.

El objetivo es que el sitio MkDocs no solo explique cómo está construido el backend, sino también cómo se planifica y sustenta la auditoría SDLC del proyecto.

## ¿Qué es la auditoría SDLC?

La auditoría SDLC es una revisión ordenada del proceso de desarrollo del software. En este caso, permite evaluar si All-InOne cuenta con evidencias suficientes sobre:

- gestión del proyecto;
- requisitos;
- arquitectura;
- desarrollo backend y frontend;
- pruebas;
- seguridad;
- documentación;
- calidad;
- mantenimiento y limitaciones.

!!! note "Enfoque académico"
    La auditoría se plantea como una evaluación académica y técnica. No representa una certificación oficial ni una garantía contractual de funcionamiento en producción.

## Relación con la documentación técnica

La auditoría utiliza como evidencia la documentación técnica del backend y frontend, los reportes de pruebas, las evidencias de calidad, Jira, GitHub, Jenkins, Swagger/OpenAPI y demás artefactos del proyecto.
